During Sprint 2, on a broad level, the team discussed:

  1. What went well in the iteration

    Getting up to speed with writing backend queries - Ian Chen
    It was relatively easy to write complex queries for creating and editing memories, including deleting replaced or removed photos from Firebase storage.

    Standardized code - Yi Sheon Tan
    We reflected upon what went wrong in Sprint 1 and took corrective actions. In the last sprint, there were many inconsistencies in coding standards and the use of packages.     Learning new approaches - Kian DSOUZA
    Learnt more efficient and new approaches for existing backend queries for display photos, as well as ease provided by using existing react packages and libraries, rather than building from scratch.

    Better decision-making processes - Calvin
    In Sprint 2, we finalised our decision-making process and applied it to every aspect of the project.

  2. What could be improved

    Reducing the duration of feature branches - Ian Chen
    It is easier to achieve our goals of CI/CD by breaking up larger features into smaller, mergeable and buildable chunks in order to minimise the pain in bringing a feature branch up to date with main, especially since APIs, types, and requirements can evolve quickly. However, since these large features are mostly complete, this will be less of a problem in the next sprint.

    UI and UX - Yi Sheon Tan
    In the current state of our application, there are various UI and UX flaws and discrepancies, such as inconsistent sizing of elements, margins and paddings. We will need to iron out these kinks in the next sprint.

    Better communication - Calvin
    It is recommended that when communicating a message or disagreement, team members can use less confronting tones.

  3. What we will commit to improving in the next iteration

    Making sure that the team is aligned on the final feature set - Ian Chen
    It is more important to produce a polished product rather than bloating additional features that users/clients will not appreciate. Furthermore, everyone should aim to flesh out the docs with diagrams and any updates, as well as write test cases/plans.

    Improving team engagement - Calvin
    In the next iteration, we will ensure that every member of the team participates in all aspects of the project, including coding, documentation and administration.
