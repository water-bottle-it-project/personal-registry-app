Team 55 IT Project : Memory view

Created by Ian Chen, last modified on Nov 10, 2022

The memory view is for viewing a single memory, which can contain many photos within it.

Anatomy

  • Title

  • Date

  • Description

  • List of tags

  • Counter for the photo number that is currently visible

  • Vertical list of photos

    • Left: photo

    • Right: info

      • Position in gallery

      • Caption

      • Timestamp (stored in db, not from reading the file on-demand)

      • Location on map - either static image or static map api

  • Scroll to top button

  • Image gallery view

    • On click: opens carousel modal with extra details

Nice-to-haves

  • Top or bottom of page: a map plotting all locations visited

    • Perhaps only visible in a drop-down menu

Design changes [End of Sprint 2]

  • No gradient

    • Focus on content

  • No persistent title with shrinking animation

    • Not a high priority

  • Combined metadata and photo info into one section

  • No individual tags for photos

  • No download, edit, delete buttons for individual photos, instead, use the unified Edit memory interface for a consistent user experience.
