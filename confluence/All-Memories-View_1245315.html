Team 55 IT Project : All Memories View

Created by Ian Chen, last modified on Oct 23, 2022

The all memories view is the main page of the site, and is the first page the user sees (if they weren’t redirected from another screen because they were not yet authenticated).

Anatomy

Titled “Your memories”. Changes to “Searched memories” if doing a search

Search UI

  • Search bar

    • Search button

    • Clear search button

  • Require clicking search button to actually execute query.

    • Just changing dates will not require clicking search.

A vertical list of memories. When tapped, we go to that specific memory’s page.

Sort orders: newest first (default), oldest first, title

  • Automatically sort without having to click confirm.

Each memory is represented as a card.

Each card has a

  • Image preview

  • Title

  • Description

    • Trim to 2 to 3 lines at most (lineClamp)

  • Date

    • Can be a range if dates different.

    • If dates same, try to condense them into a single value.

  • List of coloured tags

  • A link to view the memory, with the number of photos contained within

    • Consider counting/$size performance with MongoDB

Cards are arranged in a grid: 1, 2 or 3 wide depending on viewport width.

Scroll-to-top button in the bottom right

Nice-to-haves

  • Add tags to memories directly in timeline (a small + button)

  • Pin memories to top shelf (Favourites).

Design changes [End of Sprint 2]

  • No gradient

    • Keep the focus on the content, not the background

  • Simplify search bar: just a search box, no checkboxes

    • Users won’t want to manually specify which section to search - they may not even remember which section contained the search query!

  • No collections chips

    • Already can see memories belonging to a single memory only by going to Collections first

  • No actual timeline: so we are renaming this page from ‘Timeline’ to ‘Memories’ [still WIP in the actual deployed website].
