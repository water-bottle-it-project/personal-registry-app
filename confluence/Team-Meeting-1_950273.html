Client Meeting 1

 Discussion topics

  • Requirement of at least one tag per memory, but we propose doing an arbitrarily many tags per memory

  • Since memories could span multiple days, in the case of events, we will store both start and end dates

    • Use EXIF of contained photos to pre-fill these for the user

    • Still allow manual

  • Also store last updated date

  • Memories should be editable from the view.

  • Tags should also contain their own description too - see action item

  • Tags on memories first

    • Specific tags for photos can be added second

  • Main view is a timeline of memories by date

    • Search page can sort in different ways (TBA)

    • Search page can filter by tag or text search

 Action items

  • Excalidraw low-level designs
  • Figma high-level design including login, basic page layout for memories, photos, album and timeline
  • Discussion on tech stack
    • Either [Next.js with API routes] or [React on Vite for the frontend and Express for the backend]
  • UML diagrams for overall architecture and backend calls.
  • Start considering tag schema design in a NoSQL database (MongoDB)
